# 国际化 (i18n)

> **注意**: i18n 已前置到 Phase 1 作为基础架构的一部分。Finn Days 面向全球开发者，原生支持中英双语。
> 本文档包含完整的 i18n 实现细节，供 Phase 1 开发时参考。

## 概述

为 Finn Days 博客提供中/英双语支持，覆盖两个层面：UI 界面字符串的翻译（导航、按钮、标签等）和内容层面的多语言版本（中英文文章）。

---

## 需求分析

### 支持语言

| 语言 | 代码 | 用途 | 默认 |
| --- | --- | --- | --- |
| 简体中文 | `zh` | 主要语言，面向中文读者 | 是 |
| English | `en` | 次要语言，面向国际读者 | 否 |

### 国际化范围

1. **UI 字符串**：导航栏、页脚、按钮文本、表单标签、时间格式、错误提示等
2. **内容（文章/笔记）**：每篇文章可以有中英文两个版本
3. **URL 路由**：语言前缀路由（如 `/zh/blog/xxx`、`/en/blog/xxx`）
4. **SEO**：`hreflang` 标签、各语言的 meta 信息

---

## 技术选型：next-intl

### 库简介

[next-intl](https://next-intl-docs.vercel.app/) 是专为 Next.js App Router 设计的国际化库，由 Vercel 社区维护。

### 选型理由

| 考量维度 | next-intl 表现 |
| --- | --- |
| App Router 支持 | 一等公民支持，Server Components / Client Components 均可用 |
| 类型安全 | 完整的 TypeScript 支持，翻译键自动补全 |
| 消息格式 | 支持 ICU MessageFormat（复数、性别、嵌套等） |
| 路由策略 | 支持前缀路由、域名路由、隐式默认语言 |
| Bundle Size | 轻量，支持 tree-shaking |
| 日期/时间 | 内建日期、时间、数字的本地化格式化 |

### 安装

```bash
npm install next-intl
```

### 2. next-intl 配置

#### 请求配置

```ts
// src/i18n/request.ts
import { getRequestConfig } from "next-intl/server";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  // 验证 locale 有效性
  if (!locale || !routing.locales.includes(locale as any)) {
    locale = routing.defaultLocale;
  }

  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

#### 路由配置

```ts
// src/i18n/routing.ts
import { defineRouting } from "next-intl/routing";
import { createNavigation } from "next-intl/navigation";

export const routing = defineRouting({
  locales: ["zh", "en"],
  defaultLocale: "zh",
  localePrefix: "as-needed", // 默认语言不显示前缀
});

// 导出类型安全的导航工具
export const { Link, redirect, usePathname, useRouter } =
  createNavigation(routing);
```

// src/middleware.ts

### 4. 布局结构调整

使用 next-intl 后，需要将应用包裹在 `[locale]` 动态路由段中：

```
src/app/
├── [locale]/              # 新增：语言动态段
│   ├── layout.tsx         # 带语言的根布局
│   ├── page.tsx           # 首页
│   ├── blog/
│   │   ├── page.tsx
│   │   └── [slug]/
│   │       └── page.tsx
│   ├── notes/
│   │   └── page.tsx
│   └── ...
└── layout.tsx             # 全局布局（最小化）
```

#### 语言根布局

```tsx
// src/app/[locale]/layout.tsx
import { NextIntlClientProvider } from "next-intl";
import { getMessages } from "next-intl/server";
import { notFound } from "next/navigation";
import { routing } from "@/i18n/routing";

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;

  // 验证 locale
  if (!routing.locales.includes(locale as any)) {
    notFound();
  }

  // 加载翻译消息
  const messages = await getMessages();

  return (
    <html lang={locale} suppressHydrationWarning>
      <body>
        <NextIntlClientProvider messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

### Velite Schema 扩展

```ts
// velite.config.ts
const posts = defineCollection({
  name: "Post",
  pattern: "blog/**/*.mdx",
  schema: s.object({
    title: s.string(),
    description: s.string(),
    date: s.isodate(),
    tags: s.array(s.string()),
    // 新增：语言字段
    locale: s.enum(["zh", "en"]).default("zh"),
    // 新增：对应的其他语言版本 slug（用于语言切换链接）
    translationSlug: s.string().optional(),
    // ...
    slug: s.slug("blog"),
    content: s.mdx(),
  }),
});
```

### 按语言筛选文章

```tsx
// src/lib/content.ts
import { posts } from "#site/content"; // Velite 生成的类型安全数据

export function getPostsByLocale(locale: string) {
  return posts
    .filter((post) => post.locale === locale && !post.draft)
    .sort((a, b) => new Date(b.date).getTime() - new Date(a.date).getTime());
}

export function getTranslation(post: Post, targetLocale: string) {
  if (!post.translationSlug) return null;
  return posts.find(
    (p) =>
      p.translationSlug === post.translationSlug &&
      p.locale === targetLocale
  );
}
```

## 路由策略

### URL 结构

| 路径 | 说明 |
| --- | --- |
| `/` | 默认语言（中文）首页，等同于 `/zh`（前缀隐藏） |
| `/blog/xxx` | 默认语言的文章页 |
| `/en` | 英文首页 |
| `/en/blog/xxx` | 英文文章页 |
| `/en/notes` | 英文笔记页 |

### 默认语言处理

使用 `localePrefix: "as-needed"` 策略：

- 默认语言（中文）的 URL 不含语言前缀：`/blog/xxx`
- 非默认语言的 URL 包含前缀：`/en/blog/xxx`
- 访问 `/zh/blog/xxx` 会自动重定向到 `/blog/xxx`

## LanguageSwitcher 组件

### 设计

语言切换器放置在导航栏中，通常是一个下拉菜单或简单的切换按钮。

```tsx
// src/components/layout/language-switcher.tsx
"use client";

import { useLocale } from "next-intl";
import { usePathname, useRouter } from "@/i18n/routing";
import { Button } from "@/components/ui/button";
import { Languages } from "lucide-react";

const localeLabels: Record<string, string> = {
  zh: "中文",
  en: "EN",
};

export function LanguageSwitcher() {
  const locale = useLocale();
  const router = useRouter();
  const pathname = usePathname();

  const targetLocale = locale === "zh" ? "en" : "zh";

  const handleSwitch = () => {
    router.replace(pathname, { locale: targetLocale });
  };

  return (
    <Button
      variant="ghost"
      size="sm"
      onClick={handleSwitch}
      className="gap-1.5"
      aria-label={`切换到${localeLabels[targetLocale]}`}
    >
      <Languages className="h-4 w-4" />
      <span className="text-xs">{localeLabels[targetLocale]}</span>
    </Button>
  );
}
```

### 文章页面的语言切换

在文章详情页中，如果该文章存在对应语言版本，显示语言切换提示：

```tsx
// src/components/blog/translation-banner.tsx
import { Link } from "@/i18n/routing";

interface TranslationBannerProps {
  currentLocale: string;
  translationSlug?: string;
}

export function TranslationBanner({
  currentLocale,
  translationSlug,
}: TranslationBannerProps) {
  if (!translationSlug) return null;

  const targetLocale = currentLocale === "zh" ? "en" : "zh";
  const label =
    currentLocale === "zh"
      ? "This article is also available in English"
      : "本文也有中文版本";

  return (
    <div className="mb-6 rounded-lg border bg-muted/30 px-4 py-3 text-sm">
      <Link
        href={`/blog/${translationSlug}`}
        locale={targetLocale}
        className="text-primary hover:underline"
      >
        {label} →
      </Link>
    </div>
  );
}
```

## SEO：hreflang 标签

对于有多语言版本的页面，需要添加 `hreflang` 标签，帮助搜索引擎理解页面的语言关系。

```tsx
// src/app/[locale]/blog/[slug]/page.tsx
import type { Metadata } from "next";

export async function generateMetadata({ params }): Promise<Metadata> {
  const { locale, slug } = await params;
  const post = getPostBySlug(slug, locale);
  const translation = getTranslation(post, locale === "zh" ? "en" : "zh");

  const alternates: Metadata["alternates"] = {
    canonical: `https://finn7x.com/${locale === "zh" ? "" : `${locale}/`}blog/${slug}`,
    languages: {
      "zh-CN": `https://finn7x.com/blog/${slug}`,
    },
  };

  if (translation) {
    alternates.languages!["en-US"] = `https://finn7x.com/en/blog/${translation.slug}`;
  }

  return {
    title: post.title,
    description: post.description,
    alternates,
  };
}
```

生成的 HTML：

```html
<link rel="alternate" hreflang="zh-CN" href="https://finn7x.com/blog/getting-started" />
<link rel="alternate" hreflang="en-US" href="https://finn7x.com/en/blog/getting-started" />
```

## 日期/时间格式本地化

next-intl 提供了内建的日期时间格式化能力：

```tsx
import { useFormatter } from "next-intl";

function PostDate({ date }: { date: string }) {
  const format = useFormatter();

  return (
    <time dateTime={date}>
      {format.dateTime(new Date(date), {
        year: "numeric",
        month: "long",
        day: "numeric",
      })}
    </time>
  );
}

// 中文: "2026年3月1日"
// 英文: "March 1, 2026"
```

### 相对时间

```tsx
import { useFormatter, useNow } from "next-intl";

function RelativeTime({ date }: { date: string }) {
  const format = useFormatter();
  const now = useNow();

  return (
    <time dateTime={date}>
      {format.relativeTime(new Date(date), now)}
    </time>
  );
}

// 中文: "3天前"
// 英文: "3 days ago"
```

## 实现复杂度评估

### 复杂度矩阵

| 功能 | 复杂度 | 工作量 | 说明 |
| --- | --- | --- | --- |
| next-intl 安装配置 | 中 | 2-3h | 路由结构调整较大 |
| UI 字符串翻译 | 低 | 3-4h | 逐一翻译即可 |
| middleware 配置 | 低 | 0.5h | 标准模板 |
| LanguageSwitcher | 低 | 1h | 简单组件 |
| 内容多语言 | 高 | 4-6h | Velite schema 修改 + 筛选逻辑 |
| hreflang SEO | 中 | 2h | metadata 配置 |
| 日期本地化 | 低 | 1h | next-intl 内建 |
| 现有页面迁移 | 高 | 4-6h | 所有页面移入 `[locale]` 段 |

**预估总工作量：15-20 小时**

### 实施建议

> i18n 已前置到 Phase 1，以下为推荐的实施顺序（在 Phase 1 内完成）：

**Phase 1 内实施步骤：**

1. 安装 next-intl，配置路由和 middleware
2. 建立 `[locale]` 目录结构（从一开始就用，避免后期迁移）
3. 修改 Velite schema，添加 `locale` 字段
4. 提取所有 UI 字符串到 locale JSON (zh.json / en.json)
5. 实现 LanguageSwitcher 组件（集成到 Navbar）
6. 实现按语言筛选文章的逻辑
7. 日期/时间本地化

**后续阶段补充：**
- Phase 2: 搜索功能按语言过滤
- Phase 3: hreflang SEO 标签、sitemap 包含所有语言版本、RSS 按语言生成

### 注意事项

1. **从 Phase 1 开始就用 `[locale]` 路由**：避免后期破坏性迁移，这是前置 i18n 的核心价值
2. **翻译维护成本**：双语维护需要持续投入，初期可以只写中文，英文按需补充
3. **Velite 兼容性**：需确认 Velite 的 pattern 匹配是否支持带 locale 后缀的文件名
4. **静态生成**：确保 `generateStaticParams` 正确生成所有 locale + slug 组合
