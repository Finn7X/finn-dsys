# Phase 1: MVP + i18n 基础 实施计划

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 构建中英双语博客 MVP，支持 MDX 文章发布、分页列表、标签系统、暗色主题、项目展示和 RSS 订阅。

**Architecture:** 基于 Next.js 16 App Router + `[locale]` 动态路由实现 i18n，Velite 在构建时编译 MDX 为类型安全 JSON，所有页面默认 Server Component，仅交互组件标记 `'use client'`。

**Tech Stack:** Next.js 16.1.6 / TypeScript / Tailwind CSS 4 / shadcn/ui / Velite / next-intl / next-themes / rehype-pretty-code + Shiki / feed

## 关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| i18n 路由时机 | Phase 1 一开始就建立 `[locale]` 结构 | 避免后期破坏性迁移 |
| 内容文件命名 | `slug.mdx` + frontmatter `locale` 字段 | 比后缀命名更简洁，Velite pattern 更简单 |
| 首页位置 | `src/app/[locale]/page.tsx` | 所有页面统一在 `[locale]` 下 |
| 全局布局 vs locale 布局 | 字体在全局，Navbar/Footer/Theme/i18n 在 locale 布局 | 字体只需加载一次，其余需要 locale 上下文 |
| 博客列表分页 | URL searchParams（`?page=N`） | Server Component 友好，可 SSR |
| MDX 渲染 | Client Component（`useMDXComponent`） | Velite 编译为 code string，需要 `new Function()` |
| 草稿处理 | 所有查询默认排除 `draft: true` | 生产环境不泄露草稿 |
